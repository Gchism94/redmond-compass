S2 Home

  1. Pinned search + location

    Search stays one tap away even on Home. The location chip is editable; in cold start it softens to a “set your location” prompt rather than assuming. No featured slot anywhere — the feed is fully organic.

  2. Open now near you

    Location-driven in the personalized state (distances shown); town-wide in cold start (category labels instead of distance). Horizontal scroll keeps it glanceable.

  3. Follow feed → popular fallback

    The personalization core: bulletins from places you follow. With no follows yet, it becomes “Popular this week” plus a gentle nudge to follow — never an empty module.

  4. Events + news

    The content types that bring residents back between searches. Events are interest/saved-weighted when personalized, popular when cold. News is a light 1–2 item strip.

  5. Categories + recently viewed

    Category chips are a reliable doorway for everyone. Recently-viewed appears only for a returning user — so cold start simply doesn’t show it, rather than showing an empty row.

B4 Edit Listing

  1. Completeness meter

    The best free-tier engagement driver — completeness lifts discoverability, helping owner and directory alike. Surfaces the single next best action up top.

  2. Photos (MVP cap 5)

    A simple add-tile grid with a visible count. Five is plenty for a complete-looking free listing; the gallery beyond that is a Member field.

  3. Core fields — current-site parity

    Name, category, description, address+pin, phone, website, hours. Exactly what the current Redmond Compass collects — nothing heavier. This is the whole MVP business form.

  4. Amenity tags

    Selectable chips that map one-to-one to the S4 search facets — so what an owner sets here is what residents filter on. Profile and search speak the same vocabulary.

  5. Preview + grows later

    Live preview shows exactly the public S5 profile. Toggle Post-MVP: the locked Member section (story, menu, gallery) appears — previewable but off. It’s the same form component; MVP just doesn’t render the gated part.

Open questions. Home: default module order, and whether news sits above or below events · how aggressively cold-start nudges location vs respecting a skip. Edit Listing: which fields are truly required vs optional for a valid MVP listing · whether hours use a day-by-day editor or simple presets · photo cap at MVP (5 placeholder). Status: with these two, every MVP screen is now wireframed — the v1 build set is visually complete. Remaining unscoped screens (content: News/Community, Resources; plus S6 Events, S7 Saved, S8 Account) reuse patterns already drawn.
