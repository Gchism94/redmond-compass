Pattern-reuse map

None of these need bespoke design — each is assembled from components already specified. This is what makes them light to build.

S6 Events

The S4 result card with a date badge, grouped by time (Today / This weekend / Later). Quick date pills mirror S4’s quick filters. Save reuses the heart action.

S7 Saved

Tabbed list of S4 result cards with the S5 Call · Directions · Save verbs. The Rewards tab is the already-drawn loyalty wallet. Each tab’s empty state routes to discovery.

S8 Account

Standard settings rows + the interest chips from onboarding (S1) + notification toggles + the Switch-to-Business mode control. Links out to content/legal.

C Community

The Home feed pattern, filtered to articles + bulletins with type tags (the All / News / Bulletins blend). News is admin-published; bulletins come from businesses.

R Resources

A searchable, categorized list (Emergency · Government · Community). Mostly static civic data — the simplest screen, pure utility.

Screen set complete. With these five, every screen in the app is now specified or wireframed — consumer S1–S8, content (Community, Resources), and business B0–B9. Open questions are minor: whether Community defaults to All or News · Events list vs calendar default view · how Resources data is sourced/maintained (admin vs imported). Ready for: packaging starter files, design handoff to Claude Design, and build in Claude Code.
