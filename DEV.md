# Redmond Compass — App (dev notes)

Mobile-first installable PWA. Hyperlocal directory + community hub for Redmond, OR.
Built per `BUILD-BRIEF.md`; styled from `design-tokens.css`; layouts from the `*-wireframe.html` files.

## Run

```bash
npm install
npm run dev        # http://localhost:5173  (try /gallery for the component sheet)
npm run build      # tsc -b && vite build  (+ PWA service worker)
npm run typecheck
```

Primary canvas ~360–390px. Open dev tools device mode (iPhone 12/13 ≈ 390px) for the intended view.

## What's built (BUILD-BRIEF §12, steps 1–9)

- **Scaffold** — Vite + React 18 + TS (strict) + Tailwind 3 + `vite-plugin-pwa`. `design-tokens.css` mirrored into `src/styles/tokens.css` and wired into the Tailwind theme. Playfair Display (headings only) + DM Sans.
- **Design system** — shared components in `src/components` (+ `ui/`), shown at **`/gallery`**.
- **App shell** — bottom-tab nav, routing (§4), 480px content column, a11y baseline (skip link, focus rings, 44px taps), global `prefers-reduced-motion` off-switch.
- **Data layer** — swappable `DataSource` interface (`src/data/DataSource.ts`), `MockDataSource` over fictional seed data, TanStack Query hooks (`src/data/queries.ts`). Pick the backend in **one file** (`src/data/source.ts`) via `VITE_DATA_SOURCE`.
- **Supabase serving layer** (DATA-SOURCE.md, path B) — real backend: `supabase/migrations` define the 5 content types + `recommendations` + `profiles` (incl. `businesses.ghl_id`, `tier`; **no rating/boost columns**), RLS (public read; owner writes gated by ownership + tier; recommendations insert-only/positive-only; `profiles` own-row-only) + triggers mirroring `entitlements.ts` (+ `handle_new_user` auto-creates a profile on signup). `createSupabaseSource()` (`src/data/supabase/`) implements the full contract incl. **auth + profile** (email OTP, `getProfile`/`saveProfile`), ordering relevance/distance/recency only. Env layout: `.env.development.local` → local for `dev`, `.env.production.local` → hosted for `build` (see DATA-SOURCE.md). `mock` still works for dev. GHL sync is a seam only (`src/data/ghl/mapping.ts`). Verify with `scripts/rls-test.mjs` (18) + `scripts/auth-test.mjs` (22, OTP sign-in + owner path). Full schema/policies/auth in **DATA-SOURCE.md**.
- **Consumer read path** — Home (S2), Search (S3), Results (S4), Business Profile (S5), Events (S6) + detail, Community/News + article, Resources. All on mock data, no auth required.
- **Auth (JIT) + personalization (step 6)** — `SessionProvider` (`src/features/account/session.tsx`): local-first profile (saved/followed/saved-events/recently-viewed/interests/location/notification prefs) persisted to localStorage. **Real Supabase Auth wired** via the DataSource seam — passwordless **email OTP** (the `AuthSheet` does a two-step email → 6-digit-code flow; `signInWithOtp` → `verifyOtp`, no redirect, so the pending action completes in-place) **+ Google OAuth** ("Continue with Google"; the gated action is stashed as a pending intent and replayed after the redirect, so a save/follow still completes); the session carries the Supabase JWT so authed reads/writes use it. On first sign-in, the guest's local prefs **merge** into the user's `profiles` row (nothing lost). `requireAuth` raises the `AuthSheet` only when a guest taps Save/Follow/owner — never a browse gate; "Keep browsing without an account" escape preserved. **Mock keeps an instant sign-in** for dev (`VITE_DATA_SOURCE=mock`). First-launch lite **Onboarding** (location-as-benefit + interest chips, skippable). **Saved** (S7) and **Account** (S8) screens; `/login` for deep links. Home personalizes: follow-feed + recently-viewed rail.
- **Calendar integration (step 6b)** — `src/lib/calendar.ts` exports event times as **true UTC** (Redmond/Pacific wall-clock → UTC, DST-aware) so `.ics`/Google/Outlook all land on the right time. `EventCalendar` (month grid, alternate Events view via `SegmentedToggle`), `AddToCalendar` (.ics / Google / Outlook menu) on event detail + cards (Events list, Saved → Events, Results), and a bulk "Add all to calendar (.ics)" in Saved → Events. Live `webcal://` subscription feed is deferred (needs a backend edge function).
- **Owner path (step 7)** — `src/features/owner/*`: Claim/List (B0), light Owner Dashboard (B1), Edit Listing (B4, parity fields + per-day hours editor + amenity chips + completeness), Post Bulletin (B2), Submit Event (B3). Writes go through the `DataSource` (createBusiness/updateBusiness/claimBusiness/createBulletin/createEvent); `MockDataSource` persists them via a localStorage overlay so changes show across the app (and on the public profile). **Free tier only** — Member analytics/demand + Pro tools are entitlement seams (`can(tier, …)`), not rendered (no paywall §3). The free monthly bulletin cap **schedules instead of blocking** (never destroys work). Entry: Account → "Switch to Business" (JIT sign-in).
- **PWA (step 8)** — installable: polished `manifest.webmanifest` (id/scope/orientation/categories/shortcuts), real icons (192/512/maskable + apple-touch, generated by `scripts/generate-icons.mjs` from the compass mark), `theme-color`/`viewport-fit`/apple meta. Service worker (vite-plugin-pwa/Workbox) precaches the app shell + icons, `navigateFallback` to index.html for SPA deep links, Google-Fonts cache-first, and a network-first `/api/` seam for the real backend. `src/pwa/`: `useOnline` + `OfflineBanner` (cached states stay usable offline — verified the SW serves shell + deep routes offline against the production build), `useInstallPrompt` + `InstallBanner` (Home, dismissible) / `InstallRow` (Account) — Chromium native prompt, iOS "Add to Home Screen" hint. SW is disabled in dev; it builds + registers for production/preview.
- **Polish (step 9)** — routes are **code-split** (React.lazy + a Suspense `RouteFallback`); initial JS dropped from ~127→~99 KB gzip (gallery/owner/results load on demand). The heavy libs are also **vendor-split** (`vite.config.ts` manualChunks: `react-vendor` ~67, `supabase` ~55, `query` ~13 KB gzip), and the **data source loads on demand** (`source.ts` `getDataSource()` dynamic-imports the chosen source; `DataProvider` provides a `() => Promise<DataSource>` getter that query/mutation fns await). Net: the entry chunk is ~19 KB gzip and **`@supabase/supabase-js` (~55 KB gzip) is OFF the critical path** — the shell + skeletons paint first (no empty flash, since queryFns are async ⇒ `isLoading` from the start), then the source chunk loads in parallel. First-paint JS ≈100 KB gzip (≈ the step-9 budget, despite adding auth + Supabase). The >500 KB entry-chunk warning is gone. Route `errorElement` (`ErrorPage`) catches render/chunk-load failures without dead-ending. **Lighthouse (mobile): Accessibility 100, Best Practices 100, SEO 100, Performance ~81–86** (run against `vite preview`). Fixes: WCAG-AA contrast (darkened `--muted-foreground` + amber `--primary` so cream text clears 4.5:1), 12px minimum metadata, descriptive link text, `robots.txt`. CLS cut from ~0.40 → ~0.06–0.20 by stabilizing rail-card height + matching loading skeletons; the residual is `display=swap` font reflow (fix path: self-hosted fonts with metric overrides, or SSR).

- **Spanish (i18n)** — native-level, full-UI Spanish alongside English. Tiny typed layer in `src/i18n/` (no dependency): `dict.ts` holds EN + ES (ES typed against EN, so a missing translation is a compile error), `useI18n()` for components, `tGlobal()/getLocale()` for lib code (open-status, relative time, Intl dates). Auto-detects from `navigator.language` (es* → Spanish), persists (`rc.lang`), toggles in Account → Settings → Language and on the onboarding screen; `<html lang>` stays in sync. **Dynamic data (business descriptions, news bodies, event titles) renders as authored** — only UI chrome/labels translate; Phase 2 content pages get authored EN/ES versions on this infra.

- **Sheet → Supabase sync + account deletion (Stage 1)** — architecture change (2026-07-11): **the Google Sheet is the source of truth for directory data; GoHighLevel is out of the data path** (`businesses.ghl_id` retained but unused; the mapping seam was removed). `supabase/functions/sync-sheet` (Deno) pulls the Sheet via a Google **service account** (not the public CSV), validates the header row (missing-required/empty-sheet → **abort**, never a partial write), upserts on `id`, soft-unpublishes rows that left the sheet (`published=false`, never a hard delete; only rows it has synced before), logs each run to `sync_runs`, and fires the host deploy hook (debounced). Pure logic is `transform.ts`, unit-tested by `scripts/sync-sheet-test.mjs` (15/15). Migration `20260711…` adds `businesses.published` (+ `synced_at`) with a public-read RLS filter (`published or is_business_owner`) and the `sync_runs` audit table (service-role only). **Account deletion** is live: `supabase/functions/delete-account` (verify JWT → release owned listings → delete profile → `auth.admin.deleteUser`) behind **Account → Delete account** (confirm sheet, EN/ES) via `session.deleteAccount` → `DataSource.deleteAccount`. **Analytics (GA4) is deliberately NOT integrated or claimed** — the privacy page keeps "no third-party analytics" true until a tool is confirmed and shipped with consent, then the policy names it in the same commit. Operator setup + deploy steps: `supabase/functions/sync-sheet/README.md`. Acceptance gate: `node scripts/check-no-base44.mjs` (built output must be base44-free). **Not yet done (needs Greg/secrets):** Sheet header lockdown + Google service account, `supabase functions deploy` for both functions, `schedule.sql` cron, deploy-hook URL, GA4 decision, and the **hosted DB migration push** (classifier-gated — run `supabase db push --linked` yourself, or approve it).

- **Two shells, one app (Stage 1, Phase 2)** — `AppLayout` switches by breakpoint (`useIsDesktop`, ≥1024px): **AppShell** is the existing bottom-tab PWA; **WebShell** is the desktop site returning redmondcompass.com visitors know, rebuilt from the Phase 0 crawl (`migration/content/screenshots/`): white logo bar (+ primary links — a deliberate small addition), the original's cream eight-link guide row, orange **Sign in** pill + **Get the app** (Chromium native install prompt; iOS Share-sheet instructions; desktop hint), and the navy footer (tagline, Explore/Community/Contribute columns, ©, "Made with ♥", Privacy, EN/ES toggle). All routes/screens are shared; desktop-only differences: home renders the original's split hero (self-hosted mural `public/web/hero.jpg`) + quick-link tiles + 3-col feed row, results render a 3–4-col card grid, guides cap at a readable max-width, and the first-launch onboarding overlay is suppressed (mobile ritual only). The original's "Featured Businesses" section is deliberately NOT reproduced (equal ranking). Verify both shells with **`node scripts/ui-smoke.mjs`** (build first): mirrored 390px/1280px assertion sets — nav present, no horizontal overflow, EN/ES, zero console errors, results grid columns, guide measure + **content** (title-only checks mask a skeleton-stuck SPA — the prerendered head keeps the title correct), onboarding mobile-only, ≥44px taps, no-Featured guard. Point it at a deployed origin with `SMOKE_URL=https://… node scripts/ui-smoke.mjs` (pages-dev QA §6). **Host gotcha:** Cloudflare Pages 308s directory-style URLs to a trailing slash — prerender therefore emits flat `dist/<slug>.html` (clean 200 at the non-slash URL) and `GuideScreen` strips leading AND trailing slashes from its slug.

- **Content pages (Stage 1, Phase 2)** — the live site's 12 guides migrated in at their **original URLs** (`/about`, `/contact`, `/getting-settled`, `/help-essentials`, `/seasonal-safety`, `/get-outside`, `/pets`, `/senior-resources`, `/community-organizations`, `/new-to-the-area`, `/for-business-owners`, `/ember`). Each page is **data** (`src/features/guides/content/*.ts` — a typed EN + ES pair, authored bilingually) rendered by one `GuideScreen`; every guide is its own lazy chunk (~4–7 KB gz), so the copy never touches the entry bundle. Discoverable via the **Guides grid** on Resources, Account → About/Help (now in-app instead of linking to the old site), and Claim → owner tips. **Legacy Base44 URLs survive**: `/directory`→`/search`, `/News`→`/community`, `/submit-event`→`/manage/event/new`, squashed casings like `/GettingSettled` (redirect map in `src/app/pages.tsx` — host-level 301s can be generated from it at cutover); mixed-case paths render directly (router matches case-insensitively; `GuideScreen` normalizes the slug). **SEO**: per-page `<title>` + meta description (`usePageMeta`), and `npm run prerender` (after `build`) writes static HTML per guide + `sitemap.xml` into `dist/` (override the origin at cutover with `SITE_ORIGIN=…`). Ember memorial images are self-hosted (`public/guides/`) so they outlive the Base44 cancellation. The for-business-owners copy was adapted to platform truth: free listings are complete, reputation is Recommend (♥) — no reviews/stars — and ranking is equal. **`/privacy`** (Privacy & Terms, authored EN+ES — the old site had none) rounds out the set; Account → Privacy now stays in-app, so no Account row depends on the old site anymore. The policy documents actual behavior (local-first guest prefs, optional accounts, no ads/analytics/data sales) — **if behavior changes, update the page and its effective date in the same commit**.

- **Google Calendar inbound sync (Stage 1, Phase 1)** — `scripts/sync-gcal-events.mjs` mirrors the public Redmond Compass calendar (ICS feed) into `events`, one-way, keyed on `gcal_event_id`: upserts title/description/times/venue+address/status (DST-safe via the app's `eventStartToUtc`), expands recurring series, collapses the calendar's own duplicate entries, deletes **future** rows whose calendar entry disappeared, and keeps ICS-cancelled entries as `cancelled`. Rows the calendar doesn't own (`gcal_event_id IS NULL` — Base44 imports, app submissions) are never touched, and editorial additions (image/category/tags) survive refreshes. Scheduled every 6h by `.github/workflows/gcal-sync.yml` (service-role key only as a repo secret); run locally with `node scripts/sync-gcal-events.mjs` (`DRY_RUN=1` prints the plan; env vars target hosted).

## Non-negotiables held

- Browse/search free; no auth gate. Save/Follow fire **just-in-time** auth (the AuthSheet), never block browsing.
- **No stars / no rating field.** Reputation = the **Verified** badge + the **Recommend (♥)** count — positive-only: one per person, insert-only at the DB (can't be un-recommended, down-voted, or bombed), and it **never reorders results**. Wired on the Business Profile (`RecommendRow`; JIT-gated like save/follow). `verified_customer` on a recommendation stays a fast-follow seam.
- Equal ranking — no featured/promoted UI anywhere.
- Free listings read **complete** (profile shows nothing "locked"; member/Pro blocks simply don't render).
- Entitlement helper (`src/lib/entitlements.ts`) returns the Free set; deferred fields kept in the model.

## Decisions / flags

- **Map deferred** (§3). Results = list + filters; the List/Map toggle shows a "coming soon" seam.
- **Save = bookmark, Recommend = heart** (wireframes used ♥ for both; split for clarity).
- Community default tab = *All*; Events default view = *list* (both §14 open questions — easy to flip).
- **Backend undecided** (base44 vs Supabase, §2/§14) — non-blocking; implement `DataSource` + set `VITE_DATA_SOURCE`.
