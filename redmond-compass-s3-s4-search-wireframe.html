S3 Search / Explore

  1. Search bar + quick filters

    The focal element, with a voice slot (Tier-2, designed in now) and Open-now / Near-me pills for one-tap intent. Tapping any route leads to S4, pre-scoped.

  2. Browse by category

    For people who don’t know what to type — the larger half of users. A tappable grid is the backbone of the idle surface.

  3. Recent · trending · collections

    The empty state is never empty. Recents for one-tap repeat, trending for local pulse, collections (“New in town,” “Open late”) for browsable discovery.

  4. Unified autocomplete

    One box spans all five data types; each suggestion is tagged Business / Event / Community / Suggestion so a unified query stays legible. Results route to S4 sectioned the same way.

S4 Results & filters

  1. Header — count, filters, view, sort

    Active filter chips are removable in one tap so the user always sees why these results show. List ⇆ Map share one result set and filter state. Sort: relevance · distance · recommend · open now (no “rating” — there isn’t one).

  2. Unified result tabs

    All · Businesses · Events · Community — one search stays legible across types, mirroring S3’s autocomplete tags.

  3. Result card — act without opening

    Open status + closing time, distance, and a recommend count (♥, not stars). Inline Call · Directions · Save — the exact verbs from S5, so the vocabulary is learned once. Ranking is equal for all; no featured slots.

  4. Map view

    Clustered pins, “search this area” on pan, tap-to-peek a bottom card → full profile. Same query and filters as the list — switching never resets them.

  5. No-results recovery

    The screen that matters most: did-you-mean for typos, one-tap “broaden” actions (drop the tightest filter, widen radius), and suggested categories. A dead end always offers a forward path.
